# Explainable Clinical Decision System from Dialogues

This project builds a simple, explainable ML pipeline that predicts a triage decision from short patient-clinician dialogues.

## Problem
Given a text dialogue, predict one of:
- `emergency`
- `urgent`
- `routine`

The system is intentionally lightweight and explainable:
- Uses a hybrid of TF-IDF + tuned Logistic Regression + similar-case retrieval.
- Returns confidence, uncertainty, and top contributing words.
- Generates personalized action suggestions for each input.
- Includes safety overrides for critical symptoms.

## Setup

1. Create and activate a virtual environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. (Optional) Enable Gemini suggestions:

```bash
copy .env.example .env
```

Then edit `.env`:
- `USE_GEMINI=true`
- `GEMINI_API_KEY=...`
- `GEMINI_MODEL=gemini-2.5-flash`

## Train

```bash
python src/train.py --target-size 12000 --aspect-ratio 0.5 --save-augmented-csv
```

Outputs:
- `models/pipeline.joblib`
- `models/label_encoder.joblib`
- `models/metrics.json`
- `data/clinical_dialogues_12000.csv` (optional export)

## Streamlit App

```bash
streamlit run streamlit_app.py
```

The app provides:
- Predicted triage class
- Confidence, uncertainty, and per-class probabilities
- Top evidence tokens
- Similar historical cases
- Personalized action suggestion (local or Gemini, based on `.env`)
- Safety-rule override notice when triggered

## Predict

```bash
python src/predict.py --text "Patient has crushing chest pain, sweating, and shortness of breath."
```

Example output includes:
- Predicted class
- Confidence
- Top evidence words (positive contributions)
- Applied safety rule, if any

## Why this is explainable

For linear text models, each token has a learned weight per class.  
Explanation is generated by:
1. Transforming input into TF-IDF features.
2. Multiplying each present feature by class-specific coefficient.
3. Reporting top positive contributors.

## Notes

- This is a prototype for educational use, not medical advice.
- For real clinical deployment, use audited datasets, fairness checks, calibration, monitoring, and clinical validation.
